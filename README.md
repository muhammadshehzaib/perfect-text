# Perfect Text

**AI poster generator where the text is never garbled.**

Every AI image tool, ImagineArt included, mangles text on posters and logos:
melted letters, misspelled words, blurry typography. It is one of the most
common quality complaints in AI image generation, and it is a model-level
limitation that prompting alone will not fix.

Perfect Text solves it with a simple split of responsibility:

- **The AI generates the image:** the scene, the lighting, the composition,
  with empty space left for text.
- **The code locks the text:** the headline and sub-text are drawn on top with
  an HTML Canvas, so they are always sharp, correctly spelled, and editable.

The result: professional posters and social graphics with text that never
breaks, and that you can re-edit instantly without re-generating.

---

## Why this exists

Built as a feature prototype for [ImagineArt](https://www.imagine.art). It runs
on the Imagine text-to-image API and is designed to slot in as a "text-safe
poster mode", the kind of feature that turns a raw image model into something a
small business or marketer can actually ship from.

---

## Features

- **Before / After view:** generates the *same* poster twice. Once asking the
  AI to render the text itself (garbled), and once with the code-locked text.
  They are shown side by side, so the problem and the fix are visible at a glance.
- **One-click templates:** Coffee Shop, Gym Sale, Event, Product Drop. Each
  fills the whole form so a finished poster is one click away.
- **Drag-to-position text:** drag the headline anywhere on the poster (mouse
  or touch); top / center / bottom buttons act as quick presets.
- **Live text controls:** font, size, color, and readability backdrop, all
  updating instantly with no re-generation.
- **PNG export** at the model's full resolution.
- **Polished share preview:** a custom Open Graph card so the deployed link
  renders as a real product when shared.

## How it works

```
Browser  ──►  /api/generate  ──►  Imagine API (api.vyro.ai)
   ▲               (token stays server-side)        │
   └───────────────  background image  ◄────────────┘
   │
   └──►  Canvas: draw image, then draw real text on top  ──►  Download PNG
```

The Imagine API token never reaches the browser. The browser only ever talks
to this app's own `/api/generate` route. The text the user sees is real text
drawn by code, not generated by the model, so it never garbles or misspells.

## Run it locally

```bash
npm install
cp .env.example .env.local      # then paste your Imagine token into .env.local
npm run dev
```

Open <http://localhost:3000>

Get an API key at [platform.imagine.art](https://platform.imagine.art), under
API Keys. The token in `.env.local` must include the word `Bearer`:

```
IMAGINE_TOKEN="Bearer vk-..."
```

---

## Deploy

Deploys to Vercel as-is:

1. Push this repo to GitHub.
2. Import it into Vercel.
3. Add `IMAGINE_TOKEN` as an environment variable in the Vercel project
   settings. Never commit it.
4. (Optional) Set `NEXT_PUBLIC_SITE_URL` to your deployed URL so the share
   preview resolves correctly.

## A note for ImagineArt

Two documentation issues hit while building on the API:

1. The text-to-image endpoint requires a `variation=txt2img` form field that is
   **not listed** in the API reference
   ([reference.imagine.art/api-10672910](https://reference.imagine.art/api-10672910)
   documents only `prompt`, `style`, `aspect_ratio`, `seed`, and is marked
   "Modified over 1 year ago"). A request built strictly from the documented
   parameters fails with a misleading `400 "variation is a required parameter"`.
2. The API guide pages under `docs.imagine.art/image-generation-apis/` currently
   return **404 Page Not Found**.

Both are the first walls a new developer hits. They are quick to fix and have a
high impact on developer onboarding.
